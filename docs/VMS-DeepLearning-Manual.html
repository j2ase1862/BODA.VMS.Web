VMS.DeepLearning 매뉴얼

버전 1.1.0 · 2026-05-22 · 라벨링 → 학습 → 추론 통합 워크플로

1. 개요

1.1 역할

VMS.DeepLearning 은 Cognex ViDi 대체 도구로, 현장에서 수집한 이미지로 직접 라벨링 → 학습 → ONNX 변환 → VMS.VisionSetup 에서 추론 까지 한 번에 처리합니다. 별도 GPU 서버 없이 단일 PC 에서 동작 (TensorRT 가속 선택).

측면Cognex ViDiVMS.DeepLearning
라이선스유료 (도구별 라이선스)내장 (오픈소스 백엔드)
학습 백엔드전용 (블랙박스)YOLOv8 (Ultralytics) / PaddleOCR / PyTorch
추론 형식ViDi DBONNX → CPU / CUDA / TensorRT
커스텀 모델제한적Python 스크립트 수정 가능
VMS 통합별도 도구VisionSetup DetectionTool 등 직접 호출

1.2 아키텍처

  • UI (.NET 8 WPF): Views/MainWindow.xaml, MVVM with CommunityToolkit.
  • 학습 백엔드 (Python subprocess): scripts/*.py — YOLOv8 / Classifier / PaddleOCR / Anomaly. System.Diagnostics.Process 로 실행.
  • SAM Service (C# ONNX): Services/SamService.cs — MobileSAM ONNX 직접 추론.
  • Inference (C# ONNX): Services/OnnxDetectionInference.cs — 학습 직후 best.onnx 즉시 검증.
  • 데이터셋 저장: %AppData%\VMS\Datasets\.

2. 설치 / 환경 설정

2.1 .NET + Python 런타임

VMS MSI 설치 시 자동 포함:

  • .NET 8 — self-contained (MSI 가 포함).
  • OpenCvSharp4 4.11 — 이미지 I/O.
  • ONNX Runtime 1.21 — 추론 엔진. GPU 패키지 (CUDA/TRT) 별도 설치 시 가속.

학습용 Python 은 별도:

  • Python 3.10+ 권장 — python.org.
  • 설치 후 첫 학습 시 scripts/requirements.txt 자동 pip install: ultralytics, paddlepaddle, torch, onnx, onnxruntime-gpu.
  • 또는 Conda 환경 사용 가능 — Settings → DeepLearning → PythonPath 에서 지정.

2.2 TensorRT 가속 (선택)

YOLO ONNX 추론을 5~10× 가속. 자세한 설정은 VMS_TensorRT_Setup_Guide.md.

구성 요소버전비고
NVIDIA GPUCompute Capability 6.0+RTX 2000 시리즈 이상 권장
CUDA Toolkit12.4NVIDIA 공식 사이트
cuDNN9.xNVIDIA 개발자 계정 필요
TensorRT10.410.x 시리즈, .zip 압축 해제
환경 변수PATH 에 CUDA / cuDNN / TensorRT bin 폴더 추가재부팅 권장

2.3 사전 학습 모델 (MobileSAM)

SAM 세그멘테이션 라벨링에 사용. 첫 실행 시 자동 다운로드:

%LocalAppData%\BODA VISION AI\models\
  mobile_sam_encoder.onnx   (~40MB)
  mobile_sam_decoder.onnx   (~16MB)
인터넷 없는 환경: 사전 다운로드 후 위 경로에 수동 배치. 첫 라벨링 시 모델 로드 2~3초 소요 후 이후는 즉시.

3. UI 투어

3.1 메인 윈도우 레이아웃

Views/MainWindow.xaml — 3-패널 구조:

중앙 (캔버스)현재 이미지 + 라벨 (박스/폴리곤/마스크) 오버레이. SAM 모드 시 클릭 프롬프트.

• Training — 학습 시작 + 진행률 + 로그

3.2 3 모드 전환

좌상단 모드 선택기에서 라벨링/학습/추론 전환:

  • Labeling Mode — 이미지에 박스/폴리곤/클래스 부착. 데이터셋 빌딩.
  • Training Mode — 라벨링된 데이터셋으로 학습 시작. 백그라운드 진행률.
  • Inference Mode — 학습 직후 best.onnx 를 새 이미지에 적용해 검증 (v1.1 신규). 따로 VMS.VisionSetup 으로 이동할 필요 없음.

4. 라벨링

4.1 기본 흐름

  1. 좌상단 + New Dataset → 이름 + Task Type 선택 (Detection / Classification / OCR / Anomaly / Segmentation).
  2. 이미지 드래그&드롭 또는 폴더 선택으로 일괄 추가.
  3. 우측 Classes 탭에서 클래스 정의 (이름 + 색 + 단축키 1-9).
  4. 각 이미지를 클릭 → 라벨링 (Task Type 별 도구 자동 활성).
  5. Save — 라벨이 즉시 데이터셋 폴더에 저장됨.
  6. 충분히 모이면 Training Mode 전환.

4.2 Detection — 바운딩박스

YOLO 형식 (cls cx cy w h, 정규화 0~1) 으로 저장.

  • 드래그로 박스 생성. 클래스는 단축키 1-9 또는 우측 패널 더블 클릭.
  • 박스 클릭 후 핸들로 크기 조정.
  • 박스 우클릭 → Delete / Change Class.
  • 저장 위치: 
    %AppData%\VMS\Datasets\{name}\
  images\{train|val}\*.jpg
  labels\{train|val}\*.txt
  data.yaml

4.3 Classification — 폴더 기반

ImageFolder 형식 (PyTorch torchvision 표준):

%AppData%\VMS\Datasets\{name}\
  train\good\*.jpg
  train\ng_scratch\*.jpg
  train\ng_dent\*.jpg
  val\...
  • 이미지 하나당 클래스 1개 (배타적).
  • 단축키 1-9 누르면 즉시 해당 클래스로 분류 + 다음 이미지로 자동 이동.
  • 라벨링 속도 매우 빠름 — 1초당 1~2장 가능.

4.4 OCR — 텍스트 + 박스

PaddleOCR 형식. 각 이미지마다 {filename}.txt 에:

x1,y1,x2,y2,x3,y3,x4,y4,"읽을 텍스트"
x1,y1,x2,y2,x3,y3,x4,y4,"두 번째 영역"
  • 4점 다각형 (회전 텍스트 지원).
  • 각 박스 클릭 후 우측 패널 입력란에 텍스트 입력.
  • 합성 데이터 생성도 가능 — VMS_OCR_SynthData_Manual.html 참조.

4.5 Anomaly — GOOD / DEFECT

MVTec 형식. 정상 이미지만으로 학습 (NG 샘플 적어도 OK).

%AppData%\VMS\Datasets\{name}\
  train\good\*.png           ← 정상만 (보통 100~500장)
  test\good\*.png            ← 검증용 정상
  test\defect_type1\*.png    ← 결함 종류별 (선택)
  ground_truth\defect_type1\*.png   ← 픽셀 마스크 (선택)
  • 라벨 자체는 없음 — 폴더 분류만.
  • GOOD 버튼 / DEFECT 버튼 클릭으로 이미지 분류 → 자동 폴더 이동.
  • 학습 후 추론 시 픽셀 히트맵 + threshold 로 NG 픽셀 강조 표시.

4.6 Segmentation — SAM 폴리곤

YOLO-Seg 형식. 인스턴스 별 폴리곤. SAM 으로 빠르게 생성 → 5장에 1분 정도.

  1. 이미지에 좌클릭 — 전경 점 (포함하고 싶은 영역).
  2. 우클릭 — 배경 점 (제외하고 싶은 영역).
  3. SAM 이 실시간으로 마스크 → 폴리곤 표시.
  4. 마음에 들면 Enter → 라벨 확정 + 클래스 부여.
  5. 다시 시작은 Esc 또는 새 좌클릭.
SAM 은 자연 이미지로 사전 학습된 강력한 prompt 기반 모델. 산업 현장 이미지에도 대부분 잘 동작. 특이 형상은 점을 여러 개 찍어 보정.

5. 학습

우측 패널 Training 탭 → Start Training. Python 스크립트가 서브프로세스로 실행되며 stdout 을 UI 가 파싱해 진행률 표시.

5.1 stdout 프로토콜

학습 스크립트는 다음 marker 를 stdout 으로 출력. UI 는 이를 파싱해 진행률바와 로그 갱신.

Marker의미예
[EPOCH]현재 epoch / 총 epoch[EPOCH] 3/50
[LOSS]손실 값[LOSS] train=0.142 val=0.158
[PROGRESS]0~100 진행률[PROGRESS] 42
[ONNX]ONNX export 시작 → 완료 경로[ONNX] best.onnx
[DONE]학습 정상 완료[DONE]
[ERROR]오류 — UI 가 빨강 메시지로 표시[ERROR] CUDA out of memory

5.2 데이터셋 분할 (80:20)

라벨링 완료된 데이터셋을 train / val 로 자동 분할:

  • 기본 80% train / 20% val. UI 슬라이더로 70/30, 90/10 등 조정.
  • 이미지 셔플 후 분할 — 분류 데이터셋은 클래스별 비율 유지 (stratified).
  • 분할 시드 고정 가능 — 재현성을 위해.

5.3 하이퍼파라미터

UI 에서 직접 편집 (고급 사용자용):

파라미터기본값설명
Epochs50 (Detection) / 30 (Classifier) / 100 (Anomaly)학습 반복 횟수
Batch Size16 (Detection) / 32 (Classifier)GPU 메모리 OOM 시 줄임
Image Size640 (YOLO) / 224 (Classifier) / 256 (Anomaly)학습 해상도
Learning Rate0.01 (SGD) / 0.001 (Adam)YOLO 는 자동 cosine 스케줄
Pre-trainedyolov8n.pt / yolov8s.pt전이학습 시작점. n=nano, s=small, m=medium...
AugmentationautoYOLO 의 mosaic / mixup / hsv

5.4 ONNX Export

학습 완료 시 자동으로 ONNX 변환:

%AppData%\VMS\Datasets\{name}\runs\train\weights\
  best.pt        ← PyTorch 가중치 (학습용)
  best.onnx      ← ONNX 변환본 (추론용)
  results.csv    ← epoch 별 loss / mAP / lr 기록

VMS.VisionSetup 의 DetectionTool / SegmentationTool / ClassifierTool 에서 이 best.onnx 를 직접 로드.

5.5 진행률 표시

학습 중 우측 Training 탭:

  • 큰 ProgressBar (현재 epoch / 총 epoch).
  • 실시간 train / val loss 차트 (LiveCharts).
  • 예상 종료 시각 (ETA) — epoch 평균 시간 기반.
  • Stop 버튼 — 안전하게 학습 중단 (현재 epoch 까지 저장).
  • 완료 시 자동 Inference Mode 전환 옵션 (v1.1 신규).
GPU 메모리 부족 (CUDA out of memory): Batch Size 또는 Image Size 줄이기. Pre-trained 를 yolov8n (nano) 로 축소도 효과.

6. Inference 모드

6.1 학습 직후 즉시 검증 (v1.1 신규)

학습 완료 직후 별도 도구로 이동 없이 같은 UI 에서 검증 이미지에 모델 적용.

  1. 학습 완료 → "Test on new images" 버튼.
  2. 검증 이미지 폴더 선택 (val 셋 또는 별도 신규 이미지).
  3. 각 이미지에 박스/폴리곤 자동 오버레이 + 신뢰도 점수.
  4. 좌측 썸네일에서 다음 이미지로 빠르게 이동.
  5. 잘못된 예측 발견 시 즉시 라벨링 모드로 돌아가 보완 후 재학습.

6.2 Letterbox + NMS 전처리

Services/OnnxDetectionInference.cs:

  • Letterbox: 다양한 종횡비 이미지를 모델 입력 (640×640) 으로 패딩. 비율 유지 + 회색 패딩.
  • NMS (Non-Max Suppression): 같은 객체에 겹치는 박스 중 신뢰도 1위만 유지. IOU threshold 기본 0.45.
  • Confidence threshold: 기본 0.25 미만 박스는 표시 안 함. UI 슬라이더 조정 가능.

6.3 VMS.VisionSetup 연동

학습된 best.onnx 를 VMS.VisionSetup 에서 즉시 사용:

  1. VMS.VisionSetup 실행.
  2. 도구 팔레트 → Deep Learning → DetectionTool 드래그.
  3. 설정 패널 → Model Path → best.onnx 선택.
  4. Classes 자동 로드 (학습 시 정의한 클래스명).
  5. 이미지 / 카메라에 적용.

7. SAM Segmentation

7.1 MobileSAM 개요

Meta AI 의 Segment Anything Model 의 모바일 경량판. ~56MB ONNX 두 개 (Encoder 40MB + Decoder 16MB) 로 동작. CPU 에서도 1초 미만 응답. 자세한 내부 동작은 VMS_SAM_Segmentation_Manual.md.

7.2 전경/배경 프롬프트

모델은 클릭 점을 "프롬프트" 로 받아 마스크를 출력:

  • 좌클릭 (전경): 이 영역을 마스크에 포함시켜라.
  • 우클릭 (배경): 이 영역은 마스크에서 제외해라.
  • 여러 점을 찍을수록 정교한 마스크. 보통 2~5 점이면 충분.

7.3 폴리곤 확정 + 데이터셋 저장

SAM 마스크는 픽셀 단위 → 폴리곤으로 단순화 (cv2.approxPolyDP) 후 라벨로 저장:

labels\image_001.txt:
0 0.123 0.456 0.234 0.567 0.345 0.678 ...   ← cls polygon_xs polygon_ys

YOLO-Seg 형식이라 그대로 학습 가능.

SAM 라벨링 속도: 익숙해지면 이미지당 5~10초. 100장 데이터셋이 ~15분 정도. 기존 수동 폴리곤 (장당 30초~1분) 대비 3~6배 빠름.

8. Hand-Eye Calibration

8.1 개념

로봇 좌표계와 카메라 좌표계의 변환 행렬을 구함. 카메라가 보는 위치를 로봇이 정확히 집을 수 있게 함. 자세한 절차는 VMS_HandEye_Calibration_Manual.md.

8.2 절차 요약

  1. 카메라 시야에 체커보드 패턴 배치.
  2. 로봇이 여러 자세 (10~20 회) 로 패턴을 이동 → 각 자세마다 사진 1장 + 로봇 자세 저장.
  3. VMS.DeepLearning 의 HandEye Calibration 마법사 실행.
  4. 저장한 이미지 + 자세 로드 → 자동 계산 (OpenCV calibrateHandEye).
  5. 결과 행렬을 JSON 으로 저장 → VMS.VisionSetup 의 HandEyeCalibrationTool 에서 사용.

9. TensorRT 가속

9.1 왜 TensorRT

실행 백엔드YOLOv8s 추론 (640×640)비고
CPU (ONNX Runtime)~250ms현장 PC 기본
CUDA (ONNX Runtime GPU)~25ms10× 개선
TensorRT~8ms30× 개선. 실시간 라인 가능

9.2 의존성

§ 2.2 참조. 핵심:

  • NVIDIA GPU + CUDA 12.4 + cuDNN 9 + TensorRT 10.
  • ONNX Runtime Gpu 패키지 (NuGet 자동 포함).
  • PATH 에 CUDA / TRT bin 폴더.

9.3 활성화

VMS.VisionSetup 의 DetectionTool 설정 패널:

Execution Provider:
  ○ CPU
  ○ CUDA
  ● TensorRT  ← 선택
Cache Path: %LocalAppData%\BODA VISION AI\trt_cache\

첫 추론 시 TRT 엔진 빌드 (~30초~2분) → 캐시. 이후는 즉시 로드.

9.4 속도 비교 측정

VMS.VisionSetup → 도구 우측 결과 패널의 Execution Time (ms) 필드로 비교. 같은 이미지에서 Execution Provider 만 바꿔가며 측정.

TensorRT 캐시는 GPU 모델 / 드라이버 버전이 바뀌면 무효화. 캐시 폴더 지우고 재빌드.

10.3 자주 묻는 문제

Python 미설치 / 실행 안 됨

Settings → DeepLearning → PythonPath 에 C:\Python310\python.exe 같은 절대 경로 지정. Conda 환경은 C:\Users\{user}\miniconda3\envs\vms\python.exe.

학습 시작 후 진행률 0 에서 멈춤

첫 실행 시 pip install 진행 중 (수 분 소요). 로그 패널에 pip 출력 확인. 두 번째 이후는 즉시.

YOLO 학습이 너무 느림

GPU 사용 확인:

nvidia-smi

GPU 가 사용되지 않으면 PyTorch CUDA 버전 미스매치. pip install torch --index-url https://download.pytorch.org/whl/cu124 로 재설치.

CUDA out of memory

  • Batch Size 절반으로 (16 → 8).
  • Image Size 축소 (640 → 416).
  • Pre-trained 작은 모델 (yolov8s → yolov8n).
  • 다른 GPU 프로세스 종료 (브라우저 / 게임 / 다른 학습).

SAM 폴리곤이 너무 거칠다 / 너무 세밀하다

설정의 polygon_simplify_epsilon 조정 (기본 0.005 → 0.01 거칠게, 0.001 세밀 하게). UI 슬라이더 제공.

TensorRT 첫 빌드가 매우 오래 걸림

일반적 — 2~3분. 캐시 후 이후 즉시. INT8 quantization 사용 시 추가 calibration 시간.

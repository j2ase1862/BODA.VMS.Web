VMS.VisionSetup 엔지니어 매뉴얼

버전 1.1.0 · 2026-05-22 · 비전 도구 파이프라인 + 레시피 작성 워크스페이스

1. 개요

1.1 역할 / 대상 사용자

VMS.VisionSetup 은 엔지니어 (Lead / Supervisor) 가 사용하는 워크스페이스로, Cognex VisionPro 대체 도구입니다. 작업자(Operator) 가 직접 실행하는 VMS Launcher 와는 별개 — 운영 흐름과 분리된 도구 설정 / 시퀀스 작성 / 배치 테스트가 주된 목적.

역할VMS LauncherVMS.VisionSetup
대상현장 작업자엔지니어 / 반장
주된 동작WO 선택 → AUTO RUN → 검사레시피 작성 / 도구 연결 / 배치 검증
접근 권한Operator 이상Lead 이상 (Supervisor 권장)
실행 진입점시작 메뉴 BODA VMS / MSI 설치Launcher 의 사이드 패널 → External Tools → Vision Tool Setup

1.2 아키텍처

  • 플랫폼: .NET 8 WPF, net8.0-windows7.0, WinExe
  • MVVM: CommunityToolkit.Mvvm 8.4 ([ObservableProperty], [RelayCommand])
  • 비전 라이브러리: OpenCvSharp4 4.11 (Mat 기반)
  • 3D: HelixToolkit.Wpf.SharpDX (포인트클라우드 표시)
  • 딥러닝: ONNX Runtime (CPU / CUDA / TensorRT)
  • 핵심 클래스:
    • VisionService.Instance — 도구 실행 큐 + 결과 + 연결 관리 (싱글톤)
    • VisionToolBase — 모든 도구의 부모. Execute(Mat) → VisionResult
    • RecipeService.Instance — 레시피 CRUD + 카메라 매핑
    • ImageCanvas — Mat → WriteableBitmap + 상호작용 ROI

1.3 실행

  1. VMS Launcher 에서 Supervisor 로 Operator 로그인.
  2. 우측 톱니바퀴 (Settings 사이드 패널) → External Tools 섹션 → Vision Tool Setup 버튼.
  3. 새 프로세스로 VMS.VisionSetup 실행 — 메인 윈도우 표시.
  4. 또는 직접 실행: C:\Program Files\BODA VMS\VMS.VisionSetup.exe
Standalone (Web 연결 안 됨) 환경이거나 Operator 로그인이 없으면 LaunchVisionSetup 권한은 기본 허용. UserService 가 세팅된 환경은 권한 검사.

2. UI 투어 (3-패널 레이아웃)

Views/MainView.xaml 가 메인 화면. 좌 / 중앙 / 우 3-패널 구조 + 상단 메뉴바 + 하단 로그.

2.1 도구 팔레트 (좌)

  • 카테고리별 트리 (Image Processing / Pattern Matching / Blob / Measurement / ...).
  • 도구 아이콘을 드래그 → 캔버스 또는 레시피 트리에 드롭하면 도구 추가.
  • 검색 박스로 도구 빠른 검색.
  • 각 도구에 마우스 오버 시 1줄 설명 ToolTip.

2.2 캔버스 (중앙)

Controls/ImageCanvas.xaml + PointDrawingCanvas.xaml 사용.

  • 현재 로드된 이미지 표시 (OpenCv Mat → WriteableBitmap).
  • 마우스 휠 줌, 드래그 패닝.
  • 오버레이: ROI, 검출된 결과 (Pattern 매칭 위치, Blob 윤곽, Caliper 측정선 등).
  • 키보드 ←/→ 로 이미지 폴더 내 이전/다음 이미지 전환.
  • 2D 이미지 외에 3D 포인트클라우드 탭 (HelixToolkit) 도 같은 위치에서 전환.

2.3 결과 / 설정 (우)

  • 선택된 도구의 파라미터 편집 패널 ({Name}ToolSettingsViewModel).
  • 실행 결과 (Pass/Fail + 측정값 + ExecutionTime).
  • 오버레이 토글 (ROI 표시 / 결과 그래픽 표시).
  • 도구 간 Connection 편집 (Image / Coordinates / Result 3종 연결).

4. 레시피 시스템

4.1 Recipe / Step / Tool 구조

3단계 트리:

Recipe (예: "A1 — 톱니바퀴 검사")
 ├─ Step 1: 카메라 1 검사
 │   ├─ FeatureMatchTool      (패턴 검출 → Fixture)
 │   ├─ CaliperTool           (Fixture 기준 폭 측정)
 │   ├─ BlobTool              (스크래치 검출)
 │   └─ ResultTool             (AND 판정)
 │
 └─ Step 2: 카메라 2 검사 (선택)
     └─ ...
  • Recipe: 사용자 정의 이름, 버전, 카메라별 Step 매핑.
  • Step: 카메라 1대의 검사 흐름. PLC 트리거 1회 = 1 Step 실행.
  • Tool: 도구 인스턴스. Id (GUID), Name, ToolType, ROI, IsEnabled.

4.2 Tool Connection 3종

도구끼리 어떤 데이터를 주고받을지 명시. ToolConnection 클래스.

Connection의미예
Image출력 이미지 → 다음 도구 입력Grayscale → Threshold → Blob
Coordinates검출 점/영역 → 다음 도구의 좌표계 기준FeatureMatch → Caliper (Fixture)
Result성공/실패 상태 → 다음 도구의 분기Caliper Pass → ResultTool

연결은 우측 패널에서 도구를 선택 후 "Connect From" 드롭다운으로 설정. 또는 캔버스에서 도구 간 화살표 드래그.

4.3 카메라별 레시피

한 레시피는 여러 카메라 Step 을 묶어 정의 가능. 예를 들어 라인 1에 카메라 3대가 있으면 한 레시피 안에 Step 1/2/3 각각 카메라별 도구 트리를 작성. PLC 트리거가 오면 모든 Step 이 병렬 실행 후 ResultTool 들을 AND 조합.

4.4 Recipe Manager

Views/Recipe/RecipeManagerWindow.xaml. 메뉴 → Recipe → Manager.

  • 저장된 레시피 목록 (수정일 / 크기 / Step 수).
  • New / Open / Rename / Duplicate / Delete.
  • 각 레시피 클릭 시 RecipeEditorView 가 열림.

4.5 Import / Export

레시피는 JSON 으로 저장됨 (%LocalAppData%\BODA VISION AI\recipes\*.json).

  • Export — 단일 .json 으로 외부 저장 (백업 / 다른 PC 이관).
  • Import — .json 파일 선택 → 검증 후 추가. 동일 이름 충돌 시 자동 suffix.
  • Web 의 Recipes 페이지에서도 같은 레시피명을 등록하면 ParameterSync 로 파라미터값 동기화.

5. ROI 드로잉

5.1 ROI 종류

종류설명도구 활용
Rectangle축에 평행한 사각형일반 영역 제한
RectAffine회전된 사각형 (Affine)Fixture 기준 사각형 (회전 자동 추종)
Polygon다각형 — 클릭으로 정점 추가비정형 형상 검사 영역
Mask이진 마스크 — 픽셀 단위 ROISAM 으로 생성한 정밀 영역

5.2 편집 / 이동 / 회전

  • 드래그로 그리기. 그린 직후 핸들로 크기/회전 조정.
  • Esc 키로 드로잉 모드 종료.
  • 이미 그린 ROI 클릭 → 핸들 표시 → 이동/리사이즈.
  • Polygon: 정점 더블 클릭으로 추가, 우클릭으로 제거.

5.3 Coordinates 연결 (Fixture)

FeatureMatchTool 같은 검출 도구의 결과 좌표를 후속 도구의 ROI 기준점으로 사용:

  1. 도구 A (FeatureMatchTool) — 패턴 검출 + 회전각.
  2. 도구 B (CaliperTool) 의 ROI 를 RectAffine 으로 그림.
  3. 도구 B 의 Coordinates 연결을 도구 A 로 설정.
  4. 실행 시: A 가 검출한 위치/회전에 ROI 가 자동 정렬 → B 가 그 위치에서 측정.
이 패턴이 Fixture 패턴 — Cognex CogFixtureTool 의 대체. SLM Chat 에서 자동 제안받을 수 있음 (8. SLM Chat 참조).

6. Batch Test Runner

저장된 레시피를 폴더 단위 이미지에 대해 무인 실행 → 결과 CSV + 임계치 자동 조정 + 실패 분석. Views/BatchTest/BatchTestWindow.xaml.

6.1 기본 실행 흐름

  1. 메뉴 → Tools → Batch Test Runner.
  2. 상단 드롭다운에서 Recipe → Camera → Step 선택.
  3. 이미지 폴더 선택 (또는 단일 이미지 폴더 + 하위 재귀).
  4. Run Batch 버튼 클릭 → 진행률 바 표시.
  5. 완료 시 CSV 파일 자동 생성 ({폴더}_results_{타임스탬프}.csv).
처음 열었을 때 Run Batch 가 비활성 → Recipe/Camera/Step 모두 선택해야 활성. ComboBox 에 텍스트가 안 보이는 문제는 v1.1 에서 수정됨.

6.2 Auto-Tune Wizard

임계치를 수동 조정하지 않고 데이터로 자동 결정.

  1. Batch Test Runner → Auto-Tune 버튼.
  2. 마법사: Tool 선택 → Parameter 선택 → 현재 값 확인 (콤보 우측 표시).
  3. Sweep 범위 설정 (예: 0.5 ~ 0.95, step 0.05).
  4. Run Sweep 클릭 — 각 값으로 배치 재실행.
  5. 결과 그래프 (Pass rate vs 임계치) → 최적값 자동 추천 (Knee point).
  6. Apply → 레시피의 파라미터값 즉시 갱신.

6.3 Threshold Editor

도구별 임계치 Min/Max 일괄 편집. Views/BatchTest/ThresholdEditorWindow.xaml.

  • 현재 레시피의 모든 도구 + 파라미터 한눈에 표.
  • Min / Max / 현재값 / 단위 컬럼.
  • 인라인 편집 → Save 로 일괄 반영.
  • Auto-Tune 결과를 Apply 하면 여기에 자동 반영됨.

6.4 Failure Browser

배치 실행 후 실패 이미지만 모아 디버깅. Views/BatchTest/FailureBrowserWindow.xaml.

  • 실패 이미지 + 측정값 + NG 코드 표시.
  • 한 이미지를 메인 캔버스로 로드해 단일 실행 / ROI 조정 후 재테스트.
  • 여러 실패 케이스 클릭으로 빠르게 순회.

6.5 CSV 리포트

각 이미지 1행:

Filename, Overall, Time(ms), Tool1.Result, Tool1.Score, Tool2.Result, Tool2.Width, ...

측정값은 도구가 노출한 모든 수치 필드. CodeReaderTool 의 DecodeText 같은 문자열 필드도 v1.1 부터 포함됨.

7. Sequence Editor (PLC 트리거)

7.1 PLC 트리거 흐름

Views/Sequence/SequenceEditorView.xaml. 시스템 시퀀스는 AppData JSON (process_sequence.json) 에 저장.

일반적 흐름:

[PLC: TriggerSignal=ON]
   ↓
[VMS: Grab cameras (병렬)]
   ↓
[VMS: Inspect each camera (병렬)]
   ↓
[VMS: Combine results → Pass/Fail]
   ↓
[PLC: Result OUT signal (Pass=1, Fail=0)]
   ↓
[PLC: ResetSignal=ON → 다시 대기]

7.2 조건부 분기

  • 특정 결과 조건에 따라 다른 Step 으로 이동 가능 (예: NG 시 재검사 카메라 추가 실행).
  • 로봇 신호 연동: 픽업 좌표 출력, 위치 보정 신호 등.
  • 레시피 변경 트리거 (PLC 가 RecipeIndex 전송 → 해당 레시피 자동 로드).
시퀀스 변경은 VMS Launcher 의 autoProcessService 가 즉시 반영. PLC 통신 설정은 System Setup 마법사에서.

8. SLM Chat AI 도우미

8.1 기능 개요

Qwen2.5 기반 소형 언어모델 + RAG (도구 매뉴얼 인덱스). 자연어로 "톱니바퀴 외경을 측정하고 싶어" 같은 요청을 입력하면 적절한 도구 조합을 추천 + 레시피 자동 생성.

8.2 사용법

  1. 메뉴 → Tools → SLM Chat.
  2. 사이드 패널에 채팅 UI. 자연어 질문 입력.
  3. 예: "이 이미지에서 가장 큰 원의 지름을 측정해줘"
  4. 모델이 추천: Threshold → BlobTool (원형도 0.9+) → CircleFitTool
  5. "Apply" 클릭 시 현재 레시피에 자동 추가 + 도구 연결 자동 설정.

8.3 Fixture 패턴 자동 추천

v1.1 부터 비사각형 ROI 가 필요한 케이스를 자동 인식하여 Fixture (FeatureMatchTool → RectAffine ROI) 자동 추천. 신뢰도 점수도 함께 표시.

입력 예자동 추천
"회전하는 부품의 측정"FeatureMatchTool + Coordinates 연결 + RectAffine ROI
"여러 결함을 동시에 잡고 싶다"BlobTool + AnomalyTool 병렬 + ResultTool OR 조합
"바코드 + 텍스트 같이 읽기"CodeReaderTool + OcrTool + 결과 병합
신뢰도 점수가 0.7 미만이면 추천 결과를 검토 후 수동 조정 권장. 자동 튜닝 파이프라인은 v1.1 에서 강화됨 (실행 → 자동 검증 → 신뢰도 표시).

9.3 자주 묻는 문제

도구를 드래그하면 캔버스에 떨어지지 않음

현재 레시피에 카메라가 매핑되어 있어야 함. 새 레시피라면 먼저 Step 을 추가 후 도구 드래그.

Auto-Tune 실행 시 NullReferenceException

v1.0 의 알려진 이슈 — v1.1 에서 해결됨 (도구 선택 안 한 상태로 Run 누르면 발생). 업그레이드 권장.

SLM Chat 응답이 느림

모델 로드 첫 호출은 5~10초 소요. 이후는 즉시 응답. 백그라운드 워밍업 활성화는 SLM 설정에서.

3D 포인트클라우드 탭이 안 보임

3D 카메라 연결 안 됨 또는 카메라가 PointCloud 출력 모드 아님. Mech-Mind 카메라는 연결 후 자동 전환.

ONNX 추론 도구가 CPU 만 사용함

TensorRT 가속 설정 필요. VMS_TensorRT_Setup_Guide.md 참조 (CUDA 12.4 + cuDNN 9 + TensorRT 10 + 환경변수).
